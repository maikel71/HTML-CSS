Progress - Progreso

Documentación y ejemplos para utilizar las barras de progreso personalizadas de Bootstrap con soporte para barras apiladas, fondos animados y etiquetas de texto.

Nuevo marcado en v5.3.0 - Hemos dejado obsoleta la anterior estructura HTML para las barras de progreso y la hemos sustituido por una más accesible. La estructura anterior seguirá funcionando hasta la v6. Consulta los cambios en nuestra guía de migración.

How it works - Cómo funciona

Los componentes de progreso se construyen con dos elementos HTML, algo de CSS para establecer la anchura(width) y algunos atributos. No utilizamos el elemento (progress) de HTML5, lo que garantiza que puedas apilar barras de progreso, animarlas y colocar etiquetas de texto sobre ellas.

  • Utilizamos el .progress como envoltorio para indicar el valor máximo de la barra de progreso.
  • La envoltura .progress también requiere un role="progressbar" y atributos aria para hacerla accesible, incluyendo un nombre accesible (usando aria-label, aria-labelledby, o similar).
  • Utilizamos el .progress-bar interno únicamente para la barra visual y la etiqueta(label).
  • El .progress-bar requiere un estilo en línea, una clase de utilidad o CSS personalizado para establecer su anchura.
  • Proporcionamos una clase especial .progress-stacked para crear barras de progreso múltiples/apiladas.

Si juntamos todo esto, tenemos los siguientes ejemplos.

Bar sizing - Dimensionamiento de la barra

Width - Anchura

Bootstrap proporciona un puñado de utilidades para configurar el ancho. Dependiendo de tus necesidades, estas pueden ayudarte a configurar rápidamente la anchura de la .progress-bar.

Height - Altura

Sólo se establece un valor de height en el contenedor .progress, por lo que si se cambia ese valor, la .progress-bar interior cambiará automáticamente de tamaño en consecuencia.

Labels - Etiquetas

Añada etiquetas a sus barras de progreso colocando texto dentro del .progress-bar.

25%

Tenga en cuenta que, por defecto, el contenido de .progress-bar se controla con overflow: hidden, para que no se salga de la barra. Si su barra de progreso es más corta que su etiqueta, el contenido quedará tapado y puede resultar ilegible. Para cambiar este comportamiento, puede utilizar .overflow-visible desde las utilidades de overflow, pero asegúrese de definir también un color de texto explícito para que el texto siga siendo legible. Tenga en cuenta, sin embargo, que actualmente este enfoque no tiene en cuenta los modos de color.

Backgrounds - Fondos

Utilice clases de utilidad de fondo para cambiar la apariencia de las barras de progreso individuales.

Consejo de accesibilidad: el uso del color para añadir significado sólo proporciona una indicación visual, que no será transmitida a los usuarios de tecnologías de asistencia como los lectores de pantalla. Asegúrese de que el significado es obvio a partir del propio contenido (por ejemplo, el texto visible con un contraste de color suficiente ) o se incluye a través de medios alternativos, como texto adicional oculto con la clase .visually-hidden.

Si añade etiquetas a las barras de progreso con un color de fondo personalizado, asegúrese de establecer también un color de texto adecuado, para que las etiquetas sigan siendo legibles y tengan suficiente contraste.

Multiple bars - Múltiples bares

Puede incluir varios componentes de progreso dentro de un contenedor con .progress-stacked para crear una única barra de progreso apilada. Tenga en cuenta que, en este caso, el estilo para establecer el ancho visual de la barra de progreso debe aplicarse a los elementos .progress, en lugar de a los .progress-bars.

Striped - A rayas

Añade .progress-bar-striped a cualquier .progress-bar para aplicar una franja mediante gradiente CSS sobre el color de fondo de la barra de progreso.

Animated stripes - Rayas animadas

El degradado de rayas también se puede animar. Añade .progress-bar-animated a .progress-bar para animar las franjas de derecha a izquierda mediante animaciones CSS3.
