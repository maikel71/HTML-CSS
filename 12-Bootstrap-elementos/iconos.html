Bootstrap Icons - Iconos de Bootstrap

Biblioteca de iconos gratuita, de alta calidad y de código abierto con más de 2.000 iconos. Inclúyelos como quieras-SVGs, SVG sprite, o fuentes web. Úsalos con o sin Bootstrap en cualquier proyecto.

ICONOS

Install - Instalación

Los iconos de Bootstrap se publican en npm, pero también pueden descargarse manualmente si es necesario.

Package manager - Gestor de paquetes

Instala los iconos de Bootstrap -incluidos los SVG, el sprite de iconos y las fuentes de iconos- con npm o Composer. A continuación, elige cómo quieres incluir los iconos con las instrucciones de uso.

npm i bootstrap-icons

composer require twbs/bootstrap-icons

Download - Descargar

Las versiones se publican en GitHub e incluyen iconos SVG, fuentes, licencia y léame. También se incluye nuestro package.json, aunque nuestros scripts npm están disponibles principalmente para nuestros flujos de trabajo de desarrollo.

Descargar el último ZIP

CDN

Incluya la hoja de estilo de las fuentes de iconos -en el (head) de su sitio web o mediante @import en CSS- desde jsDelivr y empiece en cuestión de segundos. Consulta la documentación sobre fuentes de iconos para ver ejemplos.

(link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap-icons@1.11.0/font/bootstrap-icons.css")

@import url("https://cdn.jsdelivr.net/npm/bootstrap-icons@1.11.0/font/bootstrap-icons.css");

Usage - Utilización

Los iconos de Bootstrap son SVG, por lo que puedes incluirlos en tu HTML de varias maneras dependiendo de cómo esté configurado tu proyecto. Recomendamos utilizar una width: 1em (y opcionalmente una height: 1em) para facilitar el cambio de tamaño mediante font-size.

Embedded

Incrusta tus iconos dentro del HTML de tu página (en lugar de un archivo de imagen externo). Aquí hemos utilizado una width(anchura) y height(altura) personalizadas.

(svg xmlns="http://www.w3.org/2000/svg" width="16" height="16" fill="currentColor" class="bi bi-emoji-heart-eyes" viewBox="0 0 16 16")(path d="M8 15A7 7 0 1 1 8 1a7 7 0 0 1 0 14zm0 1A8 8 0 1 0 8 0a8 8 0 0 0 0 16z"/)(path d="M11.315 10.014a.5.5 0 0 1 .548.736A4.498 4.498 0 0 1 7.965 13a4.498 4.498 0 0 1-3.898-2.25.5.5 0 0 1 .548-.736h.005l.017.005.067.015.252.055c.215.046.515.108.857.169.693.124 1.522.242 2.152.242.63 0 1.46-.118 2.152-.242a26.58 26.58 0 0 0 1.109-.224l.067-.015.017-.004.005-.002zM4.756 4.566c.763-1.424 4.02-.12.952 3.434-4.496-1.596-2.35-4.298-.952-3.434zm6.488 0c1.398-.864 3.544 1.838-.952 3.434-3.067-3.554.19-4.858.952-3.434z"/)(/svg)

Sprite

Utiliza el sprite SVG para insertar cualquier icono a través del elemento (use). Utilice el nombre de archivo del icono como identificador del fragmento (por ejemplo, toggles es #toggles). Los sprites SVG permiten hacer referencia a un archivo externo de forma similar a un elemento (img), pero con la potencia de currentColor para facilitar la tematización.

Atención. Hay un problema con Chrome donde (use) no funciona en todos los dominios.

(svg class="bi" width="32" height="32" fill="currentColor")
(use xlink:href="bootstrap-icons.svg#heart-fill" /)
(/svg)
(svg class="bi" width="32" height="32" fill="currentColor")
(use xlink:href="bootstrap-icons.svg#toggles" /)
(/svg)
(svg class="bi" width="32" height="32" fill="currentColor")
(use xlink:href="bootstrap-icons.svg#shop" /)
(/svg)

External image - Imagen externa

Copia los SVG de los iconos de Bootstrap en el directorio que prefieras y haz referencia a ellos como si fueran imágenes normales con el elemento (img).

(img src="/assets/icons/bootstrap.svg" alt="Bootstrap" width="32" height="32")

Icon font - Fuente del icono

Las fuentes de iconos con clases para cada icono también se incluyen para los iconos de Bootstrap. Incluya las fuentes web de iconos en su página mediante CSS y, a continuación, haga referencia a los nombres de las clases según sea necesario en su HTML (por ejemplo, (i class="bi-alarm-clock")(/i)).

Utilice el font-size y el color para cambiar el aspecto del icono.

(i class="bi-alarm")(/i)

(i class="bi-alarm" style="font-size: 2rem; color: cornflowerblue;")(/i)

CSS

También puede utilizar el SVG dentro de su CSS (asegúrese de escapar cualquier carácter, como # a %23 al especificar valores de color hexadecimales). Si no se especifican dimensiones mediante width y height en el (svg), el icono ocupará todo el espacio disponible.

El atributo viewBox es necesario si desea cambiar el tamaño de los iconos con background-size. Tenga en cuenta que el atributo xmlns es obligatorio.

.bi::before { display: inline-block; content: ""; vertical-align: -.125em; background-image: url("data:image/svg+xml,(svg viewBox='0 0 16 16' fill='%23333' xmlns='http://www.w3.org/2000/svg')(path fill-rule='evenodd' d='M8 9.5a1.5 1.5 0 1 0 0-3 1.5 1.5 0 0 0 0 3z' clip-rule='evenodd'/)(/svg)"); background-repeat: no-repeat; background-size: 1rem 1rem; }

Styling - Estilismo

El color puede cambiarse estableciendo una clase .text-* o CSS personalizado:

(svg class="bi bi-exclamation-triangle text-success" width="32" height="32" fill="currentColor" viewBox="0 0 16 16" xmlns="http://www.w3.org/2000/svg") ... (/svg)

Accessibility - Accesibilidad

Para iconos puramente decorativos, añada aria-hidden="true". En caso contrario, proporcione una alternativa de texto adecuada. Dependiendo del método que utilice para añadir los iconos y de dónde los utilice (por ejemplo, como imágenes independientes o como único contenido de un botón o control similar), existen varios enfoques posibles. He aquí algunos ejemplos:

(!-- alt="..." on element --)

(img src="/assets/icons/bootstrap.svg" alt="Bootstrap" ...)

(svg class="bi" ... role="img" aria-label="Tools") (use xlink:href="bootstrap-icons.svg#tools"/) (/svg)

(!-- aria-label="..." on the control --)

(button ... aria-label="Mute") (svg class="bi bi-volume-mute-fill" aria-hidden="true" ...) ... (/svg>) (/button)

Working with SVGs - Trabajar con SVG

Es muy fácil trabajar con SVG, pero tienen algunas peculiaridades conocidas. Dadas las numerosas formas en que se pueden utilizar los SVG, no hemos incluido estos atributos y soluciones en nuestro código.

Los problemas conocidos incluyen:

  • Los SVG reciben el foco por defecto en Internet Explorer y Edge Legacy. Cuando incruste sus SVG, añada focusable="false" al elemento (svg). Más información en Stack Overflow.
  • Al utilizar SVG con elementos (img), es posible que los lectores de pantalla no los anuncien como imágenes o que omitan la imagen por completo. Incluya un role="img" adicional en el elemento (img) para evitar problemas.
  • Es posible que los sprites SVG externos no funcionen correctamente en Internet Explorer. Utilice el polyfill svg4everybody según sea necesario
