List group - Lista de grupos

Los grupos de listas son un componente flexible y potente para mostrar una serie de contenidos. Modifíquelos y amplíelos para que admitan casi cualquier contenido.

Links and buttons - Enlaces y botones

Utiliza as o buttons para crear elementos de grupos de listas accionables con estados hover, desactivado y activo añadiendo .list-group-item-action. Separamos estas pseudoclases para asegurarnos de que los grupos de listas formados por elementos no interactivos (como lis o divs) no ofrezcan la posibilidad de hacer clic o tocar.

Asegúrese de no utilizar las clases .btn estándar aquí.

Numbered - Numerado

Añada la clase modificadora .list-group-numbered (y, opcionalmente, utilice un elemento ol) para optar por elementos de grupos de listas numerados. Los números se generan mediante CSS (a diferencia del estilo predeterminado del navegador ol ) para una mejor ubicación dentro de los elementos de grupos de listas y para permitir una mejor personalización.

Los números son generados por counter-reset en el ol, y luego estilizados y colocados con un pseudo-elemento ::before en el li con counter-increment y content.

Horizontal

Añada .list-group-horizontal para cambiar el diseño de los elementos del grupo de listas de vertical a horizontal en todos los puntos de rotura. Como alternativa, elija una variante adaptable .list-group-horizontal-{sm|md|lg|xl|xxl} para que un grupo de listas sea horizontal a partir del min-width(ancho mínimo) de ese punto de rotura. Actualmente, los grupos de listas horizontales no pueden combinarse con grupos de listas rasantes.

Consejo de experto: ¿Desea que los elementos del grupo de listas tengan el mismo ancho en horizontal? Añada .flex-fill a cada elemento del grupo de listas.

Variants - Variantes

¡Atención! A partir de la versión 5.3.0, el mixin Sass list-group-item-variant() está obsoleto. Las variantes de elementos de grupos de listas ahora tienen sus variables CSS anuladas en un bucle Sass.

For links and buttons - Para enlaces y botones

Las clases contextuales también funcionan con .list-group-item-action para elementos a y button. Observe la adición de los estilos hover aquí no presentes en el ejemplo anterior. También se admite el estado .active; aplíquelo para indicar una selección activa en un elemento contextual de grupo de listas.

Consejo de accesibilidad: el uso del color para añadir significado sólo proporciona una indicación visual, que no será transmitida a los usuarios de tecnologías de asistencia como los lectores de pantalla. Asegúrese de que el significado es obvio a partir del propio contenido (por ejemplo, el texto visible) o se incluye a través de medios alternativos, como texto adicional oculto con la clase .visually-hidden.

Checkboxes and radios - Casillas de verificación y radios

Coloque las casillas de verificación y los radios de Bootstrap dentro de los elementos de grupos de listas y personalícelos según sea necesario. Puede utilizarlos sin labels, pero recuerde incluir un atributo aria-label y un valor para la accesibilidad.



Puede utilizar .stretched-link en labels para que se pueda hacer clic en todo el elemento del grupo de listas.
